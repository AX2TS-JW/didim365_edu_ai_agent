### 매매 실거래가: "일반 자료" vs "상세 자료" 차이

"일반(15126469)"과 "상세(15126468)" 두 버전이 있으며, **상세 자료가 상위호환**이므로 상세를 선택.

**공통 필드** (둘 다 있음):
거래금액, 건축년도, 법정동, 아파트명, 전용면적, 층, 계약년월일

**상세 자료에만 있는 추가 필드**:

| 추가 필드 | 설명 | 활용 가치 |
|-----------|------|----------|
| 동 정보 | 아파트 동 번호 (소유권이전등기 완료 건만) | 동별 시세 차이 분석 |
| 거래유형 | 중개거래 / 직거래 구분 | 직거래 비율 트렌드 분석 |
| 해제여부 | 거래 신고 후 해제(취소)되었는지 | **허위 거래 필터링 필수** |
| 해제사유발생일 | 해제된 경우 그 날짜 | 해제 시점 분석 |
| 중개사소재지 | 중개업소 시군구 단위 소재지 | 타지역 중개 비율 분석 |

> **핵심**: `해제여부` 필드로 취소된 거래를 걸러내야 정확한 시세 분석이 가능함.

### API 응답 형식 (XML)

data.go.kr API 대부분은 기본 XML로 응답. 두 가지 처리 방법:

```python
# 방법 1: JSON으로 요청 (일부 API 지원)
params = {"_type": "json", ...}

# 방법 2: XML 파싱 (모든 API 지원, 권장)
import xmltodict
response = httpx.get(url, params=params)
data = xmltodict.parse(response.text)  # → dict로 변환
```

> 참고: [PublicDataReader](https://github.com/WooilJeong/PublicDataReader) 라이브러리를 사용하면 API 파싱 없이 pandas DataFrame으로 바로 받을 수 있음.

## 3주차: LangGraph 설계 및 Elasticsearch 검색 고도화 (PDF 연동)

### 월 — LangGraph 상태 제어 설계

**커리큘럼 원본**: Nodes, Edges 핵심 개념 학습, create_agent 기반 루프를 StateGraph로 전환

**부동산 프로젝트 적용**:
- LangGraph 개념 가이드에서 Nodes, Edges 핵심 개념 학습
- **[Coding]** 1주차 create_agent 기반 루프를 StateGraph로 전환:
  ```python
  # 노드 설계
  parse_query    # "역삼동 vs 판교" → 유형: 비교, 지역: [역삼동, 판교]
  fetch_data     # 지역별 API/ES 호출
  analyze        # 평당가 계산, 비교 분석
  respond        # 자연어 답변 생성
  ```
- 단순 조회는 `parse→fetch→respond`, 비교 분석은 `parse→fetch(반복)→analyze→respond`로 분기

  - **국토연구원 부동산 시장 동향**
  - **한국부동산원 통계 보고서**
- PDF 파싱 → 청크 분할 → ES 인덱싱 파이프라인 구축

### 수 — 하이브리드 검색 및 RRF

**커리큘럼 원본**: BM25 + kNN 융합 RRF 알고리즘 적용

**부동산 프로젝트 적용**:
